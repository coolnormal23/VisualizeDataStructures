Array

Description

An array is a group of elements stored sequentially, and the most common data structure. Typically an array will have a set size, and all elements share the same data type (integer, string, etc.)

Elements in an array are refered to by index. In most languages, this starts with the first element being at index 0, the next at index 1, and so on.

Example Code

//creates an array of integers of size 5
int elements[5];
//creates and assigns 4 elements to the array
string words[] = {"this", "is", "an", "array"};

Vector

Description

A C++ data structure similar to an array, with dynamic sizing. Vectors offer more freedom, letting you add or remove elements anywhere, and access elements with the same syntax as a traditional array.

Example Code

//needed to create vectors
#include <vector>

//declare a vector
vector<int> example;

//inserting elements at the end
example.push_back(1);
example.push_back(2);
example.push_back(3);
example.push_back(4);
//1, 2, 3, 4

//access element directly
example[2] = 100
//1, 2, 100, 4

//remove last element
example.pop_back();
//1, 2, 100

Common Methods

push_back: adds element at the end of vector
pop_back: removes element at the end of vector
insert: insert elements before an index
erase: delete an element entirely
swap: swap 2 elements

Linked List

Description

A common linear data structure used to store elements in order regardless of their placement in memory. Linked lists consist of nodes that store data and a reference/pointer to the next node, culminating in the final node having a null pointer. This allows easy removal or insertion of elements in the middle, as the entire list need not be rearranged.

Example Code

Nodes are typically defined as a class or other abstract data type (e.g. struct) depending on language.

class Node
{
    public:
        int data;
        Node* next;
}
//add value to the end of linked list
void addNode(Node head, int value)
{
    Node newnode;
    newnode->data = value;
    newnode->next = NULL;
    //if no elements exist in linked list
    if(head == NULL)
    {
        head = newnode;
    }
    else
    {
        Node temp = head;
        while(temp->next != NULL)
        {
            //navigate through list to the end
            temp = temp->next;
        }
        //insert at end
        temp->next = newnode;
    }
}

Common Methods

insert: insert element at the end or within the linked list
remove: removes element from list
search: finds an element within the list
swap: swap 2 elements within the list

Binary Search Tree

Description

A tree data structure consisting of nodes with children. It follows 3 core principles:

1. The left child of a node is an element smaller than the parent element
2. The right child of a node is an element larger than the parent element
3. The left subtree and right subtree are also binary search trees.

Binary search trees follow these rules to enable binary searches, which allows fast operations inside the tree. Even in the worst case scenario, a binary search tree is only as slow as a singly linked list.

Example Code

Like linked list nodes, binary tree nodes are typically defined as a class or other abstract data type (e.g. struct) depending on language.

class Node
{
    public:
        int data;
        Node* left, right;
        Node(int value)
        {
            data = value;
            left = right = NULL;
        }
}
//insert node into binary search tree recursively
Node insertNode(Node root, int value)
{
    if(root == NULL)
     {
        //add root
        return new Node(value);
    }
    else if(value < root->value)
    {
        //go to left subnode
        root->left = insertNode(root->left, value);
    }
    else if(value &rt root->value)
    {
        //go to right subnode
        root->right = insertNode(root-right, value);
    }
    return root;
}

Common Methods

insert: inserts node into tree
search: searches for node in tree
remove: removes node from tree and rearranges if needed

Queue

Description

A data structure designed to mimic a real life queue, where the first element in will be the first element out of the queue (often called FIFO). Useful to store and retrieve elements in a definite order.

Example Code

//using C++'s standard queue template
#include <queue>

//declare a queue
vector<int> example;

//enqueue elements into queue
example.push(1);
example.push(2);
example.push(3);
example.push(4);
//1, 2, 3, 4

//dequeue elements
example.pop();
//2, 3, 4

Common Methods

push: sometimes called enqueue or push_back. inserts element into queue
pop: sometimes dequeue or pop_back. removes oldest added element from queue
empty: checks if queue is empty
size: gets size of queue

Stack

Description

A sister data structure to the Queue, the stack mimics a real life stack of items where the last element inserted will be the first one removed (LIFO).

Example Code

//using C++'s standard stack template
#include <stack>

//declare a stack
stack<int> example;

//insert elements onto the stack
example.push(1);
example.push(2);
example.push(3);
example.push(4);
//4
//3
//2
//1

//dequeue elements
example.pop();
//3
//2
//1

Common Methods

push: sometimes called push_back. inserts element on the top of the stack
pop: sometimes pop_back. removes top element from stack
empty: checks if stack is empty
size: gets number of elements in the stack

A Hash is much like an array, however the element's index is determined by the value of the element

A formula is used to determine this key and in our example we utilize the multiplication method.

Hash Collisions

You may have noticed that the formula includes the mod operator.

Because of this, the element, multiplied by a decimal less than 1, will eventually reset the index value. Therefore, certain values will end up at an index that already contains an element. This is called a hash collision.

On our website, you will be notified that the value at that index has changed

Some techniques exist to counteract this, such as moving the elemeent to the next available index until the whole array is filled or filling the array with vectors and pushing elements to this vector. Effectively giving the index multiple elements
